# Experiment — 2-Week Trial

**Nothing is permanent. Everything is an experiment.**

## Flow

1. **Define the experiment:**
   ```
   ## Experiment: [Name]
   **Hypothesis:** [What we think will happen]
   **Protocol:** [Exactly what Dan does, daily]
   **Floor version:** [The degraded version that still counts]
   **Success metric:** [How we know it worked — binary or simple]
   **Duration:** 14 days
   **Start date:** [today]
   **Review date:** [today + 14]
   ```

2. **Assign a coach:**
   - Habit experiment → James tracks compliance
   - Nutrition experiment → Andrew monitors
   - Reading experiment → Naval watches
   - Growth experiment → Ali encourages
   - Cross-domain → Rory oversees

3. **Write** experiment to `data/strategies/experiments.md`

4. **Auto-review at day 14:**
   When the review date arrives, coach prompts:
   ```
   Experiment "[Name]" completed its 2-week trial.

   Results:
   - Compliance: X of 14 days
   - [Success metric]: met / not met

   Verdict options:
   1. Keep it — make it a regular habit (James takes over)
   2. Modify — adjust and run another 2 weeks
   3. Drop — good data, not for you right now
   ```

## Examples

```
Experiment: Pre-Sleep Kindle
Hypothesis: Reading 5 min before sleep improves reading habit AND sleep quality
Protocol: Kindle on pillow. Phone across room. Read 5 min minimum.
Floor version: Open the Kindle and read 1 page.
Success metric: Did I open the Kindle? (binary)
Duration: 14 days
```

```
Experiment: Saturday Morning Swim
Hypothesis: Moving the Thursday swim to Saturday morning increases compliance
Protocol: Saturday 9 AM swim at [pool]. Bag packed Friday night.
Floor version: Walk to the pool. If you don't want to swim, walk home.
Success metric: Did I go? (binary)
Duration: 14 days
```

## Rules

- Always include a floor version — the 2-minute/30-second degraded version
- Success metrics must be binary for the first attempt (did/didn't)
- One experiment at a time per domain. Don't stack 5 experiments.
- Frame experiments with genuine curiosity: "Let's find out" not "You need to do this"
